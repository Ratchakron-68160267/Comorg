**บทที่ 4: หน่วยความจำแคช (Cache Memory)**


หน่วยความจำคอมพิวเตอร์เป็นส่วนประกอบที่มีความซับซ้อนและมีความหลากหลายสูงมากที่สุด

อย่างหนึ่งในสถาปัตยกรรมคอมพิวเตอร์ ในบทนี้เราจะเจาะลึกถึงกลไกของ "แคช" ซึ่งเป็นหัวใจสำคัญ

ที่ทำให้คอมพิวเตอร์ทำงานได้รวดเร็วอย่างในปัจจุบัน

**4.1 ภาพรวมระบบหน่วยความจำคอมพิวเตอร์**


แม้ว่าในเชิงแนวคิด หน่วยความจำจะเป็นเพียงที่สำหรับเก็บข้อมูล แต่ในความเป็นจริงไม่มีเทคโนโลยี

หน่วยความจำประเภทใดเพียงชนิดเดียวที่สามารถตอบสนองความต้องการด้านความจุ ความเร็ว และ

ราคาที่เหมาะสมได้พร้อมกัน


**ลักษณะสำคัญของระบบหน่วยความจำ (Characteristics of Memory Systems)**


เราสามารถจำแนกหน่วยความจำได้ตามคุณลักษณะต่างๆ ดังนี้:


  - **สถานที่ (Location):**   - **หน่วยความจำภายใน (Internal Memory):** เช่น รีจิสเตอร์ภายใน

CPU, แคช (Cache) และหน่วยความจำหลัก (RAM)


`o` **หน่วยความจำภายนอก (External Memory):** เช่น ฮาร์ดดิสก์, อุปกรณ์เก็บข้อมูล

แบบออปติคอล และเทป


  - **วิธีการเข้าถึงข้อมูล (Access Method):**


`o` **แบบลำดับ (Sequential Access):** ต้องอ่านข้อมูลเรียงตามลำดับ (เช่น เทป

แม่เหล็ก)


`o` **แบบโดยตรง (Direct Access):** เข้าถึงพื้นที่ข้อมูลโดยประมาณแล้วค้นหาต่อ (เช่น

ดิสก์)


`o` **แบบสุ่ม (Random Access):** สามารถเข้าถึงตำแหน่งใดก็ได้ด้วยเวลาที่เท่ากันเสมอ

(เช่น RAM)


`o` **แบบสัมพันธ์ (Associative Access):** ค้นหาข้อมูลจาก "เนื้อหา" แทนที่การใช้เลขที่

อยู่ (Address) ซึ่งเป็นหลักการสำคัญของแคช


**ลำดับชั้นของหน่วยความจำ (The Memory Hierarchy)**


ในการออกแบบ ผู้ออกแบบต้องเผชิญกับข้อจำกัด 3 ประการที่ขัดแย้งกัน (Trade-offs) คือ:


1. **ความเร็ว:** ยิ่งเร็ว ยิ่งมีราคาต่อบิตสูง


2. **ความจุ:** ยิ่งความจุมาก ราคาต่อบิตจะถูกลง แต่ความเร็วจะช้าลง


3. **ราคา:** งบประมาณที่จำกัดทำให้ไม่สามารถใช้หน่วยความจำที่เร็วที่สุดในปริมาณมากได้


เพื่อแก้ปัญหานี้ จึงเกิดการจัดลำดับชั้น (Hierarchy) โดยหน่วยความจำที่อยู่ระดับบนสุดจะเร็วที่สุด

แพงที่สุด และมีความจุน้อยที่สุด (เช่น Registers) ในขณะที่ระดับล่างสุดจะช้าที่สุดแต่มีความจุมาก

ที่สุด


**4.2 หลักการของหน่วยความจำแคช (Cache Memory Principles)**


หน่วยความจำแคชถูกสร้างขึ้นเพื่อแก้ปัญหาความเหลื่อมล้ำระหว่างความเร็วของโปรเซสเซอร์ (CPU)

และหน่วยความจำหลัก (Main Memory)


**โครงสร้างและการทำงาน**


แคชมีขนาดเล็กกว่าหน่วยความจำหลัก แต่มีความเร็วใกล้เคียงกับ CPU เมื่อ CPU ต้องการอ่านข้อมูล

ระบบจะตรวจสอบในแคชก่อน:


  - **Cache Hit:** หากพบข้อมูลในแคช CPU จะดึงไปใช้ได้ทันที


  - **Cache Miss:** หากไม่พบ ข้อมูลหนึ่ง "บล็อก" (Block) จากหน่วยความจำหลักจะถูกดึงมาเก็บ

ไว้ในแคช และส่งต่อให้ CPU


**หลักการความใกล้เคียงของข้อมูล (Locality of Reference):** หัวใจที่ทำให้แคชทำงานได้ผลคือ

ข้อมูลที่ถูกเรียกใช้มักจะถูกเรียกซ้ำในเวลาอันใกล้ (Temporal Locality) หรือข้อมูลที่อยู่ใกล้เคียงกัน

มักจะถูกเรียกใช้พร้อมกัน (Spatial Locality)


**4.3 องค์ประกอบของการออกแบบแคช (Elements of Cache Design)**


การออกแบบแคชมีความซับซ้อนและต้องพิจารณาปัจจัยหลายด้านเพื่อให้เกิดประสิทธิภาพสูงสุด:


**1. ฟังก์ชันการจับคู่ (Mapping Function)**


เนื่องจากแคชมีขนาดเล็กกว่า RAM จึงต้องมีวิธีระบุว่าข้อมูลบล็อกใดจะไปอยู่ที่ไหนในแคช:


  - **Direct Mapping:** บล็อกข้อมูลใน RAM จะมีตำแหน่งตายตัวในแคช ง่ายแต่เกิดการสลับ

ข้อมูลบ่อยหากข้อมูลที่ใช้บ่อยไปแย่งที่เดียวกัน


  - **Associative Mapping:** ข้อมูลสามารถวางตรงไหนก็ได้ในแคช ยืดหยุ่นสูงสุดแต่การค้นหา

ซับซ้อน


  - **Set-Associative Mapping:** แบ่งแคชเป็นเซต (Sets) และให้ข้อมูลเลือกวางภายในเซตนั้น

เป็นวิธีที่นิยมที่สุดเพราะสมดุลระหว่างความเร็วและความยืดหยุ่น


**2. อัลกอริทึมการแทนที่ (Replacement Algorithms)**


เมื่อแคชเต็ม ต้องมีวิธีเลือกข้อมูลเก่าทิ้งเพื่อใส่ข้อมูลใหม่:


  - **LRU (Least Recently Used):** ทิ้งข้อมูลที่ไม่ได้ใช้นานที่สุด (ได้รับความนิยมสูงสุด)


  - **FIFO (First-In-First-Out):** ทิ้งข้อมูลที่เข้ามาเป็นตัวแรก


  - **LFU (Least Frequently Used):** ทิ้งข้อมูลที่ถูกเรียกใช้น้อยที่สุด


**3. นโยบายการเขียน (Write Policy)**


เมื่อมีการแก้ไขข้อมูลในแคช ข้อมูลใน RAM ต้องถูกอัปเดตด้วย:


  - **Write-Through:** เขียนลงแคชและ RAM พร้อมกัน ข้อมูลตรงกันเสมอแต่ช้า


  - **Write-Back:** เขียนลงแคชก่อน จะเขียนลง RAM ก็ต่อเมื่อบล็อกนั้นถูกเตะออกจากแคช เร็ว

กว่าแต่ซับซ้อนกว่าในการรักษาความถูกต้องข้อมูล


**4.4 การจัดระเบียบแคชของ Pentium 4**


ในยุคของ Pentium 4 สถาปัตยกรรมแคชมีความซับซ้อนมากขึ้นเพื่อรองรับการประมวลผล

ประสิทธิภาพสูง:


  - **Multilevel Caches:** เริ่มมีการใช้แคชหลายระดับ (L1, L2, L3)


  - **Split Cache:** L1 มักจะถูกแบ่งเป็น **Instruction Cache** (สำหรับคำสั่ง) และ **Data**

**Cache** (สำหรับข้อมูล) เพื่อให้ CPU สามารถดึงคำสั่งและข้อมูลได้พร้อมกันโดยไม่แย่งบัส

กันเอง


**4.5 คำศัพท์สำคัญและคำถามทบทวน**

**คำศัพท์สำคัญที่ควรทราบ:**


  - **Hit Ratio:** อัตราส่วนความสำเร็จในการพบข้อมูลในแคช


  - **Miss Penalty:** เวลาที่เสียไปเมื่อไม่พบข้อมูลในแคชและต้องไปดึงจาก RAM


  - **Unified Cache:** แคชที่เก็บทั้งข้อมูลและคำสั่งรวมกันในที่เดียว
